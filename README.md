# 🐀 VerminGame

Welcome to VerminGame — a small experimental game project where I explore game development, coding patterns, and creative systems.

This project is not about building a finished product (yet).  
It's about learning by building.

## 🎯 Goals of the Project

The main purpose of this project is to improve my skills as a developer by experimenting with:

- 🧠 Game logic and system design  
- 🎮 Player movement and animation systems  
- ⏱️ Synchronizing animations with code using timers and update loops  
- 🧩 Structuring code in a clean and scalable way  
- 🎨 Learning how gameplay design and code interact

Instead of following tutorials step-by-step, I try to solve problems myself, debug systems, and gradually improve the architecture as the project grows.

## 🛠 What I'm Currently Exploring

Some of the systems I'm experimenting with right now include:

- Directional sprite systems (16 directions)
- Sprite animation loops driven by code
- Movement and physics integration
- Animation timing and frame control
- Debugging workflows in Unity
- Organizing game logic into maintainable scripts

## 🧪 Why This Project Exists

I believe the best way to learn programming is to build real systems, break them, and rebuild them better.

This repository acts as a playground for experimentation, where I test ideas, implement mechanics, and learn how different parts of a game interact.

## 🚧 Work In Progress

Expect messy commits, refactors, and experiments.

That's part of the process.
